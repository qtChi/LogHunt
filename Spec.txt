## 4. Storage Architecture

This section resolves the DuckDB read-only consistency issue. Two storage backends are used with strictly separated responsibilities.

### 4.1 DuckDB — Analytical Read Layer

DuckDB is used exclusively for analytical queries over Parquet partitions. It operates in read-only mode on the Parquet store. It never stores application state, metadata, or baseline data. This claim is absolute — no component writes to DuckDB except `ParquetWriter`, which writes Parquet files directly via `pyarrow` without going through `DuckDBLayer`.

`DuckDBLayer` exposes only `SELECT` queries. The read-only enforcement has two layers: the DuckDB connection is opened with `read_only=True`, which is a driver-level enforcement, not a string scan. A secondary string scan for forbidden keywords exists as defense-in-depth only and is documented as such — it does not substitute for the connection-level guarantee. Both layers are documented in `docs/ARCHITECTURE.md`.

**DuckDB manages:**
- Parquet analytical queries (all five OCSF event class partitions)
- Replay Parquet queries (when explicitly requested via session flag)

### 4.2 SQLite — Metadata and Audit Layer

SQLite manages all persistent application state. A single `metadata.db` file stores all mutable application metadata. The `AuditLogger` writes to this database. `BaselineEngine` writes baseline data to this database. `SigmaEngine` writes the rule store to this database.

**SQLite manages:**
- Query audit log
- Ingest audit log
- Rule audit log and rule store
- Computed baselines
- Metric snapshots

This separation is clean: DuckDB is read-only over log data. SQLite is read-write over application metadata. No component crosses this boundary.

### 4.3 Replay Partition Isolation

Replay partitions are physically isolated from production partitions under `{base_path}/replay.parquet/session_id={id}/`. The isolation rules are explicit and enforced at the `DuckDBLayer` level:

- `DuckDBLayer.execute_query` never scans replay partitions unless `include_replay=True` and a `session_id` are explicitly passed.
- `QueryBuilder.execute` never passes `include_replay=True` under any circumstances. Production queries are always replay-isolated.
- `SigmaEngine.backtest_rule` passes `include_replay=False` for production backtests and `include_replay=True` with a `session_id` only when called from `ReplayEngine.test_rule_against_session`.
- `ReplayEngine.test_rule_against_session` is the only public method that produces a replay-scoped backtest.

This means an analyst using the normal investigation UI cannot accidentally query replay data. Replay data only surfaces when explicitly testing a rule against a named replay session.

## 6. Specification of Class `OCSFFieldRegistry`

**Build Priority: Phase 1 — must exist before OCSFEvent**

`OCSFFieldRegistry` is the central schema authority for all OCSF field definitions. It is the single source of truth for which fields exist, their types, whether they are required, and which event classes they belong to. Both `OCSFEvent` and `OCSFNormalizer` depend on it. It is loaded once at application startup from a versioned JSON schema file.

**Test file:** `tests/unit/test_ocsf_field_registry.py`

---

#### `OCSFFieldRegistry(schema_path: str)` — Constructor

Loads the OCSF field schema from a JSON file at `schema_path`. The schema defines all valid fields across all supported event classes. Raises `FileNotFoundError` if path does not exist. Raises `ValueError` if the schema JSON is malformed or missing required structure. The registry is immutable after construction — no fields can be added or removed at runtime.

---

#### `get_field_definition(field_path: str) -> FieldDefinition | None`

Returns the `FieldDefinition` for a given dot-notation field path such as `"actor.user.name"`. Returns `None` if the field is not defined in the registry. Never raises on unknown field — `None` is the correct signal for unknown fields and is used by `OCSFEvent` constructor validation.

---

#### `get_fields_for_class(class_uid: int) -> list[FieldDefinition]`

Returns all field definitions applicable to a given event class, including both universal fields and class-specific fields. Raises `ValueError` if `class_uid` is not a supported class. Returns a non-empty list — all supported classes have at least the six universal required fields.

---

#### `get_required_fields(class_uid: int) -> list[str]`

Returns the dot-notation paths of all required fields for a given event class. Raises `ValueError` if `class_uid` is not supported.

---

#### `is_valid_field(field_path: str, class_uid: int) -> bool`

Returns `True` if the field path is a valid, registered field for the given event class. Returns `False` otherwise. Never raises — unknown fields and unsupported classes both return `False`.

---

#### `get_field_type(field_path: str) -> str | None`

Returns the declared type string for a field such as `"VARCHAR"`, `"INTEGER"`, `"TIMESTAMP"`, `"VARCHAR[]"`. Returns `None` if the field is not in the registry.

---

#### `FieldDefinition` — Data Class

A `FieldDefinition` contains: `field_path: str`, `field_type: str`, `required: bool`, `applicable_classes: list[int]`, `description: str`, `source_examples: list[str]`. This is a read-only data class — no setters.

## 8. Specification of Class `OCSFEvent`

**Build Priority: Phase 1**

`OCSFEvent` is the canonical normalized data object produced by all parsers and consumed by storage, querying, and detection components. It validates all fields against `OCSFFieldRegistry` at construction time. Unknown fields are rejected, not silently stored.

---

#### `get_class_uid() -> int`

Returns `class_uid`.

---

#### `get_time() -> datetime`

Returns the UTC `datetime`.

---

#### `__eq__(other: object) -> bool`

Equal if and only if `class_uid`, `time`, `metadata_log_source`, and `metadata_original_time` all match. Returns `False` for non-`OCSFEvent` comparisons — never raises.

#### `__repr__() -> str`

Returns `OCSFEvent(class_uid=6003, source=evtx, time=2026-03-01T14:32:11Z)`.

**Build Priority: Phase 1**

#### `DuckDBLayer(parquet_base_path: str)` — Constructor

#### `execute_query(sql: str, params: dict, include_replay: bool = False, replay_session_id: str | None = None) -> list[dict]`

Executes a parameterized SELECT query. The DuckDB connection is opened `read_only=True` — write statements will fail at the driver level regardless of this method's content. A secondary keyword scan for `INSERT`, `UPDATE`, `DELETE`, `DROP`, `CREATE` exists as defense-in-depth and is documented as supplementary only — it does not replace the connection-level guarantee. When `include_replay=False` (the default), replay partition directories are excluded from the scan path. When `include_replay=True`, `replay_session_id` must also be provided — only the specified session partition is included. `QueryBuilder` never passes `include_replay=True`. Only `SigmaEngine` and `ReplayEngine` may pass `include_replay=True`. Returns results as list of row dictionaries. Returns empty list for zero results. Raises `TypeError` if `sql` or `params` is `None`. Raises `ValueError` if `sql` is empty, if `include_replay=True` but `replay_session_id` is `None` or empty.

---

#### `get_schema(class_uid: int) -> dict`

Returns field name to DuckDB type string for the given class partition. Returns empty dict if no partition exists. Raises `ValueError` if `class_uid` is unsupported.

---

#### `partition_exists(class_uid: int, date: str) -> bool`

Returns `True` if the partition exists for the given class and `YYYY-MM-DD` date. Raises `TypeError` if either argument is `None`. Raises `ValueError` if `date` is not a valid date string.

---

#### `close() -> None`

Closes the connection. Subsequent `execute_query` calls raise `RuntimeError`. Multiple `close()` calls are safe no-ops.

---

## 14. Specification of Class `QueryIntent` and Nested Models

**Build Priority: Phase 2**

`QueryIntent` is a Pydantic v2 model. It is the only object `QueryBuilder` accepts. Produced exclusively by `IntentExtractor`.

**Test file:** `tests/unit/test_query_intent.py`

---

#### `QueryIntent(event_class, time_range, filters, group_by, order_by, order_dir, limit, include_fields)` — Pydantic Model

`event_class` must be one of `{1001, 3001, 3002, 4001, 6003}`. `time_range` must be a `TimeRange` with `end > start`. `filters` must be a list of `FieldFilter` objects each passing allowlist validation for the event class. `group_by`, `order_by`, `include_fields` must all be in the class allowlist when provided. `limit` must be 1–5000. `order_dir` must be `"asc"` or `"desc"`. Empty `filters` is valid. Raises `ValidationError` on any violation.

---

#### `FieldFilter(field, operator, value)` — Pydantic Model

`field` must be in the class allowlist. `operator` must be one of `{"eq", "neq", "gt", "lt", "gte", "lte", "contains", "startswith", "in", "not_in", "is_null", "is_not_null"}`. `value` must be type-compatible with the field's declared type — integer fields reject string values even if they represent valid numbers, IP fields validated via `ipaddress` module, timestamps validated as ISO 8601. `value` must be `None` only for `is_null` and `is_not_null`. `in` and `not_in` require non-empty list. Raises `ValidationError` on any violation.

---

#### `TimeRange(start, end)` — Pydantic Model

Both must be timezone-aware `datetime` objects. `end` must be strictly after `start`. Maximum 90-day range. Raises `ValidationError` on violation.

---

#### `to_sql_params() -> tuple[str, dict]`

Returns `(sql_template, params_dict)`. All values in params dict — none interpolated into SQL. Raises `RuntimeError` if called pre-validation (cannot occur under normal usage).

---

#### `get_event_class() -> int`

Returns `event_class`.

---

#### `get_filters() -> list[FieldFilter]`

Returns filters list. Returns empty list if none specified.

## 27. Unspecified Behavior — Judgment Decisions

All decisions below are documented in `DECISIONS.md` in the repository root.

**Empty string vs. None.** All methods treat empty strings and whitespace-only strings identically to `None` for required string arguments. Void methods and constructors raise `ValueError`. Non-void methods return `None` or `False`.

**Duplicate events on ingest.** Both events are written. Deduplication is query-time, not ingest-time. Full source fidelity is preserved.

**LLM unavailability.** If Ollama is not running, `IntentExtractor.extract` and `AnomalyExplainer.explain` return `None` and log the failure. The rest of the application is fully functional. UI surfaces a warning.

**Baseline below min_observations.** `compute_baseline` returns `None`. A partial baseline is more dangerous than no baseline. Suppression is correct.

**Rule versioning.** `update_rule` never deletes prior versions. All versions are preserved in SQLite. `get_all_rules` returns latest version per rule by default.

**MITRE pattern matching on absent fields.** A `None` field never satisfies a condition. Absence of a field is not a wildcard.

**CoverageAnalyzer with no confirmed rules.** `compute_coverage` returns `coverage_percentage=0.0` with all techniques in `gap_techniques`. Valid state on fresh installation.

**IOCMatcher row cap.** If events exceed `max_enrich_rows`, only the first `max_enrich_rows` are enriched. The cap is logged as a warning. Analyst can lower their query limit or raise the cap in Settings.

**DuckDB read-only enforcement.** The primary enforcement is the `read_only=True` connection flag at the driver level. The secondary keyword string scan is defense-in-depth and is documented as supplementary only. Both are described in `docs/LLM_SECURITY_MODEL.md` and `docs/ARCHITECTURE.md`.
